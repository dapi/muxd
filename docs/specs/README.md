# Specs

Implementation-ready feature specs live in this directory.

Each spec should describe one delivery slice with:

- scope
- requirements
- acceptance criteria
- edge cases
- linked ADRs and plans

Naming rule:

- `docs/specs/YYYY-MM-DD-topic-name.md`

Start from:

- `docs/templates/spec-template.md`

There are no approved implementation-ready specs yet. The current repo is still closing the stack decision and documentation structure.
